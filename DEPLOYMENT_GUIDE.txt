## TROUBLESHOOTING

### Issue: Port 5000 already in use
Solution: Edit app.py last line
  Change: app.run(debug=True, host='0.0.0.0', port=5000)
  To:     app.run(debug=True, host='0.0.0.0', port=5001)
  Then restart

### Issue: Module not found errors
Solution: Reinstall dependencies
  python -m pip uninstall -r requirements.txt -y
  python -m pip install -r requirements.txt

### Issue: Excel file rejected
Solution: Verify file format
  - Must be .xlsx or .xls format
  - Must have MPN and Qty columns
  - No merged cells or formatting
  - File size < 16MB

### Issue: Calculation errors
Solution: Check inventory file
  - Column names are case-insensitive
  - Remove blank rows/columns
  - Ensure Qty column has numbers
  - Check for special characters in MPN

### Issue: Website won't load
Solution: Check server is running
  - Terminal should show: "Running on http://127.0.0.1:5000"
  - Try http://127.0.0.1:5000 instead of localhost
  - Check browser console for JavaScript errors (F12)

## PRODUCTION DEPLOYMENT

For production use (not development):

1. Install Gunicorn:
   pip install gunicorn

2. Run production server:
   gunicorn -w 4 -b 0.0.0.0:5000 app:app

3. Use reverse proxy (Nginx/Apache):
   - Handle SSL/TLS
   - Manage multiple workers
   - Serve static files efficiently

4. Set Flask debug=False in app.py

5. Use environment variables for configuration

6. Consider adding database for persistence

## SECURITY NOTES

Development Server:
- Single-threaded
- Not suitable for production
- Debug mode enabled
- No SSL/TLS

Production Recommendations:
- Use production WSGI server (Gunicorn, uWSGI)
- Enable SSL/TLS via reverse proxy
- Add authentication if needed
- Validate all file uploads properly (already done)
- Rate limiting for API
